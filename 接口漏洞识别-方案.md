- 一般情况下未授权访问，是指某个接口未做鉴权处理

> 正常情况下我们识别未授权可能是某个请求包(get/post加上json格式)，响应包通常为json、xml格式
>
> - get:
>
> 1. 请求包
>
> ```

# 越权访问

- 越权访问一般	情况下是指的有token进行访问接口，然而接口为进行权限划分所导致的超越当前用户的权限访问特定接口。

> 识别的核心在于访问id，或者权限划分Id，参数用于权限划分

正常情况下我们识别越权可能是某个请求包(get/post加上json格式)，响应包通常为json、xml格式，偶尔在地址相关接口可能会存在html形式的越权（越权查看他人信息）		

**效率与成本平衡**

大模型 API 调用有一定的成本和延迟，可以考虑在二次校验前，先用轻量的规则引擎过滤掉明显的误报，只对高可疑的片段调用大模型，这样能大幅提升效率。

**上下文关联能力**

可以尝试把 JS 文件和对应的请求上下文（如请求 URL、响应头）一起喂给大模型，这样模型能更准确地判断敏感信息的 “泄露场景”，比如是调试日志残留，还是配置文件硬编码。

# AI-二次-检测敏感信息泄漏

**HTML/JS/API的响应中可能泄露敏感信息(如OSS存储桶密钥泄露、小程序AppSecert泄露等...)。通过AI技术或定制化的检测逻辑,可以智能分析响应内容,及时发现并预警敏感信息泄漏,降低数据泄露风险。

针对这些，可以采用二次校验的形式进行分辨

针对敏感信息所存在的片段->若存在手机号，真实的key

​	--> 进一步跟踪相关js文件

​		--> 调用大模型API

​			---> 识别该js文件，整合相关所可能存在的敏感信息

​					---> 将大模型所整理的信息，形成卡片式信息聚合（便于用户进行人工分析）

​																																									^	

​																																									|

​																																									|

​			---> 在敏感信息界面进行增加ai提取控件，点击后大模型可根据对应用户指定目标js进行整合(形成卡片式)



**效率与成本平衡**

大模型 API 调用有一定的成本和延迟，可以考虑在二次校验前，先用轻量的规则引擎过滤掉明显的误报，只对高可疑的片段调用大模型，这样能大幅提升效率。
